## ③ 엣지케이스 검토 (과장 여부 점검)

1. **M-03 "전 환경 미표시" (유일한 과장)**: 폰트 cap height(digits ≈ 0.70em) 기준 숫자 상단
   y≈103.8 < 112 → viewBox 안. 데스크톱(스케일 1.3)에서 약 10px 높이의 잘린 숫자 상단부가
   렌더된다. 즉 "완전 소실"이 아니라 "55% 클리핑되어 판독 불가". 심각도(HIGH)·수정안·근본 원인은
   그대로 유효하므로 실무 영향 없는 표현 과장.
2. **M-02의 뷰포트 일반화**: 계산상 절단 임계 뷰포트 ≈ 390px. 412px 기기에선 현재가 절단이
   발생하지 않고 바도 22px 폭으로 부분 회복된다. 스크린샷 기기(1080px/DPR3 = 360 CSS px)에서는
   완전 재현되므로 판정은 유지하되, "360–412px" 범위 전체로 일반화한 서술은 미세한 부정확.
3. **M-06 배경색 가정**: 모든 대비를 surface(#1e293b) 기준으로 계산했으나 stat-block 등 일부는
   #0f172a 위(→ #64748b는 3.75:1). 어느 배경 기준으로도 AA 미달이므로 결론 불변.
4. **심각도 배정 점검**: CRITICAL 2건(M-01 정보 화면 밖+전면 가로 스크롤, M-02 시각화 전면 소실)
   은 "정보 판독 불가" 정의와 일치. M-03을 HIGH로 둔 것도 타당 — 헤더 pill(47.3·중립)로 동일
   정보가 접근 가능해 "판독 불가" 등급까진 아님. M-05를 HIGH(오독 유발)로 둔 것 역시 오분류가
   투자 판단을 왜곡할 수 있어 적절. 등급 인플레이션 징후 없음.
5. **로컬 vs 배포본 캐비앳**: 리포트가 스스로 데이터 차이(46.6 vs 47.3)를 공개하고 구조 동일성을
   레이아웃 패턴으로 논증한 점은 정직. 리뷰어도 S1~S5의 grid 구조·클리핑 위치·컬럼 붕괴 패턴이
   로컬 코드 산출과 1:1 대응함을 확인 — 구조적 결론에 영향 없다는 주장 성립.
