## 2. Five-Why Root Cause

| Layer | Why | Evidence |
|---|---|---|
| 1. Surface | Parser pulled `target_price` from `chartData2` historical monthly series (latest non-null = 2026-05-29) instead of static table (current = 2026-06-29). | `tools/consensus/naver_parser.py` pre-fix: `parse_chart_data2` returned series; `parse_opinion_and_analysts` only grabbed cells[0] and cells[-1] (opinion + analysts), discarding cells[1] (target), [2] (EPS), [3] (PER). |
| 2. Design choice | I chose chartData2 first because it was JSON-clean and easy to parse. Static-table cell extraction was added only for opinion/analyst-count. | Code order in commit: parse_chart_data2 -> derive_target_price_trend -> parse_opinion_and_analysts (limited scope). No "current snapshot" extraction. |
| 3. Why static value dismissed | Earlier inspection saw `3,177,083` in the static table but I (PM Agent) judged it "too high" based on outdated SK-hynix-price-around-200K-won prior. Marked it unsuitable, kept chart series. | Verbal commentary in Phase 14-0-B2 thread ("number looks unrealistic"). No external anchor consulted before dismissal. |
| 4. Why cross-validation missed it | X1-X7 verified self-consistency: agent A's output == agent B's re-derivation of the same source. X3 even ran a second regex on chartData2 -- same source -> same answer. No test compared against an external anchor (user, FDR, arithmetic invariant). | `xv_x3_validation_vs_analysis.txt` shows both paths read `chartData2` and reproduced 42.08%. PASS but FALSE POSITIVE. |
| 5. Why no external anchor was required | The cross-validation matrix I designed had only intra-system tests. There was no PER * EPS = close invariant, no FDR cross-source check, no "two-source rule" for any field. | `scripts/cross_validate_phase14.py` pre-fix had only X1-X7. |

## 3. Why the user-provided ground truth was decisive

`PER * EPS = close` is a closed-form arithmetic identity that the page itself
publishes. With three numbers (PER=8.54, EPS=307,655, close=2,628,000) any one
can be verified from the other two. Because all three were in the same HTML
fixture, the discrepancy was always **provable from the raw response** -- we
simply did not check.

This is the canonical pattern of "missing arithmetic anchor": when a system
holds an algebraic invariant in its data, omitting the invariant check
leaves the system vulnerable to silent semantic errors.

## 4. Fixes Applied (committed to working tree, not yet to repo)

### Fix-1: `parse_static_consensus_table()` in `tools/consensus/naver_parser.py`
- Strict 5-cell extraction in header order:
  `[opinion, target_price, eps, per, n_analysts]`
- Returns dict with `target_price`, `eps`, `per`, `n_analysts`,
  `investment_opinion`, plus `raw_cells` for audit.

### Fix-2: Multi-source reconciliation in `parse_wisereport_html()`
- Returns BOTH `static_target_price` and `chart_latest_target_price`.
- `latest_target_price` defaults to static (authoritative current).
- `prior_target_price` defaults to chart[-1] (one month ago).
- New `reconciliation` dict records:
  - `per_times_eps` and `close_latest` and their `diff_pct`
  - `static_target` and `chart_latest_target` and their `diff_pct`
- Warnings emitted if PER * EPS != close within 1%, or static-chart gap > 50%.

### Fix-3: Analyzer schema bump to v0.2
- `raw_inputs` now includes `static_target_price`, `static_eps`, `static_per`,
  `close_price_latest`, `chart_latest_target_price`, `target_price_change_label`.
- Q1 change derived from static-vs-chart, not chart-vs-chart.

### Fix-4: New cross-validation tests
- **X8 (arithmetic invariant)**: PER * EPS ~= close, within 1%.
  Anchor: closed-form identity in the data.
- **X9 (external anchor)**: FinanceDataReader.DataReader('000660',...)
  latest close ~= extracted close, within 5%.
  Anchor: independent data source (KOSPI official feed).

### Fix-5: Renderer adds "산술 일관성 점검" block
- Shows EPS, PER, PER*EPS, close, diff%, and static-vs-chart gap.
- User can sanity-check by eye without running tests.

### Fix-6: `CLAUDE.md` Things to Avoid -> OL-7
- Anti-confirmation-bias rule on dismissing extracted data.
- Multi-source requirement when same field appears multiple times.
- Arithmetic invariant requirement when PER/EPS/close co-extracted.
- External-anchor requirement in cross-validation matrices.

## 6. What this incident teaches about cross-validation

Before: "68/68 unit tests PASS + 7 cross-validation gates" felt strong.
Reality: All 7 gates probed self-consistency. Zero gates probed correctness
against an external anchor. **Self-consistency PASS does not imply
correctness.**

The cure is not more tests -- it is **a different kind of test**:
- Internal: agent A's output == agent B's recomputation. (We had 7.)
- **Arithmetic anchor**: data must satisfy a closed-form identity present
  in the source. (We had 0; now 1.)
- **External anchor**: extracted value must agree with an independent
  source. (We had 0; now 1.)

OL-7 requires at least one of each type when the data permits.
