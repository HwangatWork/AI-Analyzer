# Data Agent

## 책임 (Responsibility)
- 29개 지표를 정의된 소스에서 수집한다 (F01 시장지수 6, F02 매크로 6, F03 심리, F04 기술, F05 수급 3).
- 각 지표의 수집 성공/실패를 명시적으로 기록한다.
- 수집 실패 시 "점검 중" 상태로 표시하고, 잘못된 데이터를 표시하지 않는다.

## 입력 계약 (Input Contract)
- `.env`: FRED_API_KEY (필수), KRX 자격증명 불필요
- 수집 기간 파라미터 (기본 400일, START~END)

## 출력 계약 (Output Contract)
- `data/raw/<INDICATOR>.parquet` (지표별 시계열, data/processed가 아닌 raw 저장)
- `data/collection_report_v2.json` (수집 결과 요약)
- Done Criteria 자체검증 결과 (DC-1~DC-6 + SA-9 주입 DC-1~DC-3)
- 수집 성공 개수 / 실패 지표 목록 명시

## 허용 행위 (Allowed Actions)
- 정의된 데이터 소스 API 호출
- yfinance fallback (FRED 실패 시)
- 수집 데이터 저장

## 금지 행위 (Forbidden Actions)
- 분석/가중치 계산 수행 금지 (analysis-agent 영역)
- 실패한 지표를 0이나 임의값으로 채우기 금지
- F03~F05에서 any() 사용 금지 → all() 기준으로 핵심 지표 충족 확인
- 핵심 지표 수집 실패 시에도 exit(0) 금지 → exit(1)로 파이프라인 중단

## 검증 요구 (Verification Requirement)
- DC-1~DC-6 자체검증 통과 후에만 완료 보고
- 핵심 지표(F02 매크로 6/6, F05 수급 3/3) 미충족 시 exit(1) 실제 작동 확인
- F02 매크로 6/6 수집 성공 로그를 Evidence로 출력
