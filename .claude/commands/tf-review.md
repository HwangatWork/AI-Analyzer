# /tf-review ‚Äî Team Framework Peer Review

You will orchestrate a Team Framework peer review. The user-provided proposal
text or file path is in `$ARGUMENTS`.

The framework's two hooks (`tf_schema_check`, `tf_aggregate`) are dormant until
the `.active` flag is set. Follow Steps 1-5 strictly in order.

## Step 1 ‚Äî Pre-flight: set .active flag

Generate a timestamp session ID and create the gate flag:

```bash
python -c "from agents.tf_active import set_active; import time; sid = time.strftime('%Y%m%d_%H%M%S'); print(set_active(sid)); print('TF session: ' + sid)"
```

Note the session ID ‚Äî needed for output path and Step 5.

## Step 2 ‚Äî Save proposal

Write the proposal text to `output/peer_review/<session_id>/proposal.md`.
This is the canonical input the 13 reviewers will reference.

## Step 3 ‚Äî Spawn 13 reviewer subagents (parallel, single message)

Use the Task tool **13 times in ONE message** (parallel execution per
Claude Code docs) with these `subagent_type` values:

- `data-agent`
- `analysis-agent`
- `evaluator-agent`
- `validation-agent`
- `decision-agent`
- `stock-agent`
- `sector-agent`
- `news-agent`
- `narrative-agent`
- `ui-agent`
- `report-agent`
- `audit-agent`
- `meta-audit-agent`

Each Task prompt template (substitute proposal text + agent role):

> Review this PM proposal from your **{agent-domain}** perspective and respond
> with a **single JSON object** matching `schemas/peer_review_response.schema.json`:
> required fields = `agent`, `domain_relevance` (direct|indirect|none),
> `agreement` (agree|partial|disagree), `urgency_vote` (1-99 integer),
> `reason` (20-600 chars, Korean OK). Optional: `addition` object with
> `file` + `change` (‚â•10 chars) + `function` (nullable).
>
> Your domain failure modes: see your MD's "Peer Review Concerns" section
> (Phase 13-B-4 future).
>
> Proposal:
> [paste the saved proposal content from Step 2]

The SubagentStop hook (`tf_schema_check.py`) validates each response against
schema; exit 2 forces a teammate to revise if invalid. The PostToolBatch hook
(`tf_aggregate.py`) auto-aggregates all 13 responses into `aggregate.md` once
the batch resolves.

## Step 4 ‚Äî Post-flight: clear .active flag

Critical: must run regardless of Step 3 outcome (use try/finally semantics
in your reasoning). Failure to clear leaves hooks active for non-TF future
Task batches.

```bash
python -c "from agents.tf_active import clear_active; clear_active(); print('TF session cleared')"
```

## Step 5 ‚Äî Read aggregate + summarize

Read `output/peer_review/<session_id>/aggregate.md` (auto-generated by
PostToolBatch hook). Summarize for the user:

1. Consensus matrix (1 line per agent)
2. Urgency revote winner
3. New items surfaced (any addition that ‚â•3 agents endorsed)
4. Meta-Patterns section (if present ‚Äî high-signal cross-domain consensus)
5. Minority Dissent section (if present ‚Äî direct-relevance dissenter
   worth surfacing)

If aggregate.md is missing, report which step failed.
