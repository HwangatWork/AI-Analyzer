# AI Analyzer — 실전 예측력 검증 및 개선 (Phase P: Purpose Return)

## 목적
이 프로젝트의 존재 이유는 "29개 시장 지표 → S&P500/KOSPI 매수/매도 의사결정 자동화"다.
인프라(파이프라인·검증 체계)는 이미 완성됐다. 이번 작업은 **산출물 자체의 예측력**을
측정하고 개선하는 것이다. 인프라 신규 구축은 금지한다 (기존 도구 재사용만 허용).

## 3대 산출물별 검증 과제

### P-1. 지표 가중치 랭킹 — 예측력 백테스트
- 질문: "가중치 상위 지표가 실제로 하위 지표보다 지수 방향을 더 잘 예측하는가?"
- 방법: walk-forward 백테스트. 과거 시점 T에서 그 시점까지 데이터만으로 랭킹 산출
  (PIT invariant: as_of ≤ T, 미래 데이터 누출 금지) → T+1~T+20일 지수 방향 적중률을
  상위 5개 vs 하위 5개 지표 그룹으로 비교.
- 성공 기준: 상위 그룹 적중률 > 하위 그룹 + 통계적 유의성 (p < 0.05, 최소 30개 시점).
- 실패 시: 가중치 산식(시차 상관·Granger·동행 페널티)의 어느 항이 노이즈인지 분해 분석.

### P-2. 지수 기여 Top5 — 사후 정확도 측정
- 질문: "Top5로 선정된 종목이 실제 지수 변동 기여 상위였는가?"
- 방법: 과거 산출 이력(output/ 히스토리, git log의 daily pipeline commits)에서
  각 날짜의 Top5 예측을 복원 → 해당일 실제 지수 기여도(시총가중 수익률)와 대조
  → precision@5 산출.
- 성공 기준: precision@5 ≥ 0.6 (무작위 기대치 대비 명확한 우위).

### P-3. 수혜 종목 Top5 — forward return 검증
- 질문: "수혜 Top5가 이후 실제로 시장을 이겼는가?"
- 방법: 각 산출 시점 이후 5/10/20일 forward return을 벤치마크(S&P500, KOSPI) 대비 측정.
- 성공 기준: 20일 초과수익 평균 > 0, 승률 > 50%.

### P-4. BUY/SELL/HOLD 시그널 — 최종 성과 측정
- 질문: "decision.json 시그널을 그대로 따랐다면 buy-and-hold보다 나았는가?"
- 방법: 시그널 이력 기반 단순 시뮬레이션 (BUY=보유, SELL/WAIT=현금).
  거래비용 0.1% 가정. Sharpe·MDD·누적수익 3개 지표로 buy-and-hold와 비교.
- 성공 기준: 정직한 보고. 이겼으면 근거 수치, 졌으면 패인 분해 (진입 지연? 지표 노이즈?).

## 실행 원칙 (CLAUDE.md 영구 룰 준수)
1. **사전 논의 필수**: 구현 전 analysis-agent + evaluator-agent + audit-agent 가
   검증 방법론(백테스트 설계의 lookahead 위험, 표본 크기, 벤치마크 선정)을 peer review.
   방법론 합의 없이 코드 작성 금지.
2. **3-tier cross-validation**: 각 P 항목 완료 시 audit + meta-audit + evaluator 독립 검증.
   PASS 표기는 검증 통과 후에만.
3. **Level 8+ Evidence**: 실제 수치·exit code·로그. "검증했다" 서술만 = 미완료.
4. **정직한 보고**: 예측력이 없다고 나오면 그게 이 작업의 가장 가치 있는 발견이다.
   성공 기준 미달을 숨기거나 기준을 사후 완화하는 것 금지 (FP-001 패턴).
5. **회귀 유지**: pytest agents/tests/ 기존 baseline 유지 + 각 P 항목당 회귀 테스트 추가.
6. **인프라 작업 금지**: 새 workflow·hook·검증 스크립트 신설 금지.
   기존 데이터(output/, data/, git 히스토리)와 기존 도구만 사용.
